# Dataset Tools 개요

## 프로젝트 목적

RIDE 프로젝트의 멀티센서 데이터셋(카메라, LiDAR, Radar, GNSS, Perception)을 효율적으로 처리하기 위한 통합 도구 모음입니다.

## 해결하려는 문제

### 1. 데이터 분산 및 일관성 부족
- 기존에는 `time_align/`과 `scripts/`에 변환 스크립트가 분산되어 있음
- 공통 기능이 중복 구현되어 유지보수 어려움
- 일관되지 않은 CLI 인터페이스

### 2. 센서 간 시간 동기화 복잡성
- 각 센서마다 다른 샘플링 레이트 (LiDAR 10Hz, GNSS 100Hz 등)
- Raw 데이터의 타임스탬프가 정렬되지 않음
- 프레임 단위 매칭 알고리즘 필요

### 3. 데이터셋 포맷 변환
- Raw 데이터를 학습용 Source 포맷으로 변환 필요
- 데이터셋 버전 마이그레이션 (v1.0 → v1.2)
- Calibration 값 통합 및 변환

## 주요 기능

### Time Alignment (시간 정렬)
- **재조립 (Reassemble)**: LiDAR 프레임 기준으로 모든 센서 데이터를 시간 정렬하여 재구성
- **검증 (Verify)**: 동기화 결과의 정확성 검증 및 통계 생성
- **리포트 (CSV Generation)**: 센서별 시간 차이 및 매칭 결과를 CSV로 출력

### Data Conversion (데이터 변환)
- **Raw → Source**: AVI/JSONL 통합 파일을 프레임 단위 PNG/JSON으로 분할
- **포맷 마이그레이션**: 데이터셋 구조 버전 업그레이드
- **Calibration 변환**: 센서 캘리브레이션 값 포맷 통일
- **Radar 처리**: H5 파일에서 scan/track 데이터 추출 및 절대 속도 계산

## 주요 사용자

- **자율주행 데이터 엔지니어**: 데이터셋 전처리 및 품질 검증
- **ML 엔지니어**: 학습용 데이터셋 준비
- **연구원**: 센서 융합 데이터 분석

## 기대 효과

1. **개발 효율성 향상**: 통합된 CLI로 일관된 사용 경험
2. **유지보수 용이**: 모듈화된 구조로 기능 추가/수정 간편
3. **데이터 품질 개선**: 자동화된 검증 및 리포트 기능
4. **재사용성 증가**: 공통 유틸리티를 라이브러리로 활용 가능
